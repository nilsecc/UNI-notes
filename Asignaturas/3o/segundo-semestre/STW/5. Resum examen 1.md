## 💻 Parte 2: Fundamentos de JavaScript (Variables y Scope)

- **Ámbitos (Scopes)**:
    
    - **Global**: No recomendado; accesible en todo el script.
        
    - **Function Scope (`var`)**: La variable es visible dentro de la función, pero no respeta bloques `{}` internos.
        
    - **Block Scope (`let` / `const`)**: El estándar moderno. La variable solo existe dentro del bloque de llaves `{}` donde se declara.
        
- **Mutabilidad de `const`**: Aunque no se puede reasignar una variable `const`, si apunta a un objeto o array, sus propiedades internas sí pueden modificarse.
    

---

## 🧠 Parte 3: Conceptos Avanzados y "Star Moments"

- **Closures (Clausuras)**: Es la relación entre una función y el entorno léxico (variables externas) donde fue creada.
    
    - Permite que una función "recuerde" variables incluso después de que la función padre haya terminado su ejecución.
        
- **Contexto `this`**:
    
    - En funciones estándar (`function`), `this` depende de quién llama a la función.
        
    - En **Arrow Functions (`=>`)**, `this` no existe como tal; se hereda del contexto exterior (evita el "shadowing" de `this`).
        

---

## 🚀 Parte 4: Node.js y Asincronía

- **Funcionamiento**: Node.js es de un solo hilo (single thread).
    
- **Gestión de tareas**:
    
    1. Las funciones asíncronas salen de la **Call Stack** y pasan a la **Web API**.
        
    2. Al terminar, la Web API envía el callback a la **Callback Queue**.
        
    3. El **Event Loop** monitoriza la Call Stack; cuando está vacía, mueve las tareas de la cola a la pila para ejecutarlas.

let b = null;

const unitAdder = (startBy) => {
    return () => ++startBy; // Clausura: recuerda 'startBy'
};
const adder = unitAdder(5);
console.log(adder()); // 6

// === 4. CLASES (Syntactic Sugar) ===

    #count = 0; // Propiedad privada

    getCount() { return this.#count; }
}
