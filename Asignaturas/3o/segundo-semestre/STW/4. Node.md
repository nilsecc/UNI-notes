## Com va js?
Les funcions asincrones es treuen de la call stack y se les pasa a la web API (es single threat sino es quedaria esperant). **Tot lo asincron de la call stack es pasa a la web API** 
- La web API acaba la operacio asincrona y fa crica a una funcio callback que es pasa a la callback queuedins l'event loop que fa un while true i mira si te coses a fer
- Quan la call stack queda buida es pasan de la event loop a la call stack
